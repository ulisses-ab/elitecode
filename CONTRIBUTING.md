# Contributing

Contributions are welcome. Please follow these steps:

1. Fork the repo and create a branch from `main`.
2. Follow the local setup instructions in `README.md`.
3. Make your changes. Keep commits focused — one logical change per commit.
4. Open a pull request describing what you changed and why.

## Adding problems

See `CLAUDE.md` for the full problem authoring spec — it covers the directory layout, runner protocol, test format, and how to seed problems into the database.

## Reporting bugs

Open an issue with steps to reproduce, expected behaviour, and actual behaviour.

## Security vulnerabilities

Please do **not** open a public issue for security vulnerabilities. See `SECURITY.md` instead.
